pyspike.distances.isi_distance(spikes1, spikes2, interval=None)[source]¶

Computes the isi-distance I of the given spike trains. The isi-distance is the integral over the isi distance profile \(S_{isi}(t)\):

\[I = \int_{T_0}^{T_1} S_{isi}(t) dt.\]
Parameters:
  • spikes1 – ordered array of spike times with auxiliary spikes.
  • spikes2 – ordered array of spike times with auxiliary spikes.
  • interval (Pair of floats or None.) – averaging interval given as a pair of floats (T0, T1), if None the average over the whole function is computed.
Returns:

The isi-distance I.
Return type:

double

pyspike.distances.spike_sync(spikes1, spikes2, interval=None)[source]¶

Computes the spike synchronization value SYNC of the given spike trains. The spike synchronization value is the computed as the total number of coincidences divided by the total number of spikes:

\[SYNC = \sum_n C_n / N.\]

  • spikes2 – ordered array of spike times with auxiliary spikes.
  • interval (Pair of floats or None.) – averaging interval given as a pair of floats (T0, T1), if None the average over the whole function is computed.

pyspike.distances.spike_sync_profile(spikes1, spikes2)[source]¶

Computes the spike-synchronization profile S_sync(t) of the two given spike trains. Returns the profile as a DiscreteFunction object. The S_sync values are either 1 or 0, indicating the presence or absence of a coincidence. The spike trains are expected to have auxiliary spikes at the beginning and end of the interval. Use the function add_auxiliary_spikes to add those spikes to the spike train.

Parameters:
  • spikes1 – ordered array of spike times with auxiliary spikes.
  • spikes2 – ordered array of spike times with auxiliary spikes.
Returns:

The spike-distance profile \(S_{sync}(t)\).
Return type:

pyspike.distances.spike_sync_profile_multi(spike_trains, indices=None)[source]¶

Computes the multi-variate spike synchronization profile for a set of spike trains. For each spike in the set of spike trains, the multi-variate profile is defined as the number of coincidences divided by the number of spike trains pairs involving the spike train of containing this spike, which is the number of spike trains minus one (N-1).

Parameters:
  • spike_trains – list of spike trains
  • indices (list or None) – list of indices defining which spike trains to use, if None all given spike trains are used (default=None)
Returns:

The multi-variate spike sync profile \(<S_{sync}>(t)\)
Return type:

pyspike.spikes.load_spike_trains_from_txt(file_name, time_interval=None, separator=' ', comment='#', is_sorted=False)[source]¶

Loads a number of spike trains from a text file. Each line of the text file should contain one spike train as a sequence of spike times separated by separator. Empty lines as well as lines starting with comment are neglected. The time_interval represents the start and the end of the spike trains and it is used to add auxiliary spikes at the beginning and end of each spike train. However, if time_interval == None, no auxiliary spikes are added, but note that the Spike and ISI distance both require auxiliary spikes.

Parameters:
  • file_name – The name of the text file.
  • time_interval – A pair (T_start, T_end) of values representing the start and end time of the spike train measurement or a single value representing the end time, the T_start is then assuemd as 0. Auxiliary spikes will be added to the spike train at the beginning and end of this interval.
  • separator – The character used to seprate the values in the text file
  • comment – Lines starting with this character are ignored.
  • sort – If true, the spike times are order via np.sort, default=True
Returns:

list of spike trains
